# 赛道数据规范（schema）

一份合格的赛道分析 = **5 个模块**。下面逐个说明字段含义、颗粒度要求、常见坑。
最终把它们组织成一个 JSON（见文末），喂给 `scripts/build_track_excel.py` 生成 Excel。

> 贯穿原则：**对标《清洁工具》黄金样例的深度**——每一项都要"具体到能直接指导投放"，
> 拒绝"高质量、性价比、好用"这种谁都能说的空泛词。

---

## 模块① metrics · 赛道整体指标

本期该赛道创意的均值指标。有报表就填真实数字，没有就沿用/估算并标注。

| 字段 | 含义 | 单位 | 示例 |
|------|------|------|------|
| `creativeCount` | 本期创意条数 | 条 | 156 |
| `ctr` | 点击率 | % | 4.8 |
| `play3s` | 3 秒完播率 | % | 58 |
| `cvr` | 转化率 | % | 6.3 |
| `cpm` | 千次曝光成本 | 元 | 71 |

---

## 模块② sellingWords · 卖点词云

这个赛道**消费者最买账的卖点**，做成词云（权重越大，网页上字越大越居中）。

- 数量：**12-16 个**（少于 10 个显得单薄）。
- `word`：卖点词，尽量短平快、有画面感（如"免手洗""一擦即净""360°旋转"），
  避免长句、避免抽象（"质量好"❌ → "吸水不掉毛"✅）。
- `weight`：0-100 相对权重。**第一个最核心的给 100**，其余按重要度递减，
  不要都给一样的值（词云会失去层次）。

```
{ "word": "免手洗", "weight": 100 }
```

---

## 模块③ painWords · 用户痛点词（可选但强烈建议）

消费者**没被满足时的痛**，与卖点一一呼应（有痛点才有卖点）。

- 数量：4-6 个。
- 同样口语化、有画面感："拖地弯腰累""抹布越擦越脏""油污擦不掉"。
- 结构同卖点词：`{ "word": "...", "weight": 0-100 }`。

---

## 模块④ sellingContext · 卖点动因 · 需求背景【最重要】

**这是整份分析最有价值的部分**：不是罗列卖点，而是回答"这个卖点**为什么此刻成立**"。
把"时令 / 换季 / 场景 / 情绪"等**背景动因** → 转成**真实需求** → 落到**支撑卖点词**。

每条 3 个字段：
| 字段 | 含义 | 要求 |
|------|------|------|
| `driver` | 触发动因（背景） | 具体到时节/场景/情绪，如"三伏厨房·重油污爆发" |
| `need` | 由此产生的真实需求 | 一句话说清消费者此刻想要什么 |
| `words` | 支撑卖点词（数组） | 从模块②里挑能落地这个动因的词 |

- 数量：**2-3 条**，覆盖当期主要的 2-3 个购买场景。
- 反例（太空）：`driver:"夏天到了"` → 正例：`driver:"梅雨/回南天·地面返潮"`。

```
{
  "driver": "三伏厨房·重油污爆发",
  "need": "高温下厨房油烟重、油污顽固，想要强力去污、不脏手",
  "words": ["厨房重油污", "强力去污", "不脏手"]
}
```

---

## 模块⑤ scripts · 常见爆款话术

**能直接抄去投放**的高转化口播/字幕，来自≥2 条跑量素材里反复出现的说法。

- 数量：6 条左右。
- 口语、短、有转化力："免手洗，脏水一按就走""厨房油污一擦就掉，不费劲"。
- 就是一个字符串数组。

---

## 模块⑥ keyPoints · 跑量关键点（固定 4 维度）

这个赛道**怎么打才跑得动**，固定从 4 个维度描述，`title` 用下面固定值，`desc` 写具体打法：

| title（固定） | desc 要回答 |
|------|------|
| `钩子/强对比` | 开场 3 秒怎么抓眼球？用什么视觉冲击/前后对比？ |
| `节点/季节痛点` | 绑定哪个季节/节点？放大什么场景痛点？ |
| `福利/紧迫` | 价格锚点怎么给？用什么制造紧迫（限时/立减/赠品）？ |
| `明星/IP` | 有无明星/达人/IP 背书？信任链怎么建（真实测评 vs 明星代言）？ |

```
{ "title": "钩子/强对比", "desc": "开场用『糖浆一拖即净』强视觉冲击，挤水前后地面反差极大..." }
```

---

## 最终输入格式：track JSON

把 6 个模块拼成一个对象（`painWords` 可省略）。`name` 必须是网页里已有的赛道名。
存成 `inbox/XX赛道.track.json`，再运行 `scripts/merge_into_creative.py` **直写进 `site/data/creative.js`**：

```bash
python3 scripts/merge_into_creative.py inbox/清洁工具.track.json          # 写入本地数据源
python3 scripts/merge_into_creative.py inbox/清洁工具.track.json --push    # 写入并推送上线
```

> 合并策略：按 `name` 覆盖本文件定义的 6 个模块；`topMaterials` 等网页侧字段**原样保留**。
> （如仍想走 Excel 回收流程，可改用 `scripts/build_track_excel.py` 生成《填报-XX赛道.xlsx》。）

```json
{
  "name": "生活日用-清洁工具",
  "metrics": { "creativeCount": 156, "ctr": 4.8, "play3s": 58, "cvr": 6.3, "cpm": 71 },
  "sellingWords": [
    { "word": "免手洗", "weight": 100 },
    { "word": "一擦即净", "weight": 94 }
  ],
  "painWords": [
    { "word": "拖地弯腰累", "weight": 90 }
  ],
  "sellingContext": [
    { "driver": "三伏厨房·重油污爆发",
      "need": "高温下厨房油烟重、油污顽固，想要强力去污、不脏手",
      "words": ["厨房重油污", "强力去污", "不脏手"] }
  ],
  "scripts": [
    "免手洗，脏水一按就走",
    "厨房油污一擦就掉，不费劲"
  ],
  "keyPoints": [
    { "title": "钩子/强对比", "desc": "..." },
    { "title": "节点/季节痛点", "desc": "..." },
    { "title": "福利/紧迫", "desc": "..." },
    { "title": "明星/IP", "desc": "..." }
  ]
}
```

> 完整可跑通的示例见 `scripts/example-清洁工具.json`（可直接用 `merge_into_creative.py` 写入验证）。
