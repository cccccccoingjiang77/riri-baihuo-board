# 日用百货 · 对客宣导 OnePage

面向客户的一页式行业宣导看板。外层豆腐块展示核心信息，点击下钻查看详细报表。

## 目录结构

```
site/
├── index.html          # 总览层（两大豆腐块）
├── selection.html      # 选品看板（分链路 + 分周期节气）
├── creative.html       # 卖点&创意分析看板（分赛道 → 词云/话术/跑量关键点/Top素材）
├── assets/
│   └── style.css       # 全局样式（深色专业质感）
├── data/               # ★ 多人协作填充区（只改这里，不用碰页面逻辑）
│   ├── selection.js    # 选品数据（由脚本从选品台xlsx生成，可手工再编辑）
│   ├── creative.js     # 卖点&创意数据（5个赛道）
│   └── calendar.js     # 年度节气/热点周期日历
└── tools/
    └── build_selection_data.py  # 从选品台xlsx重新生成 selection.js
```

## 本地预览

```bash
cd site
python3 -m http.server 8770
# 浏览器打开 http://localhost:8770/index.html
```

## 数据更新

### 选品数据（selection.js）
选品台 xlsx 更新后，重新生成：
```bash
python3 tools/build_selection_data.py
```
> 注意：原始 xlsx 未逐条标注「全域通 / ADQ」渠道，脚本按链路推荐字段做启发式划分。
> 如需精确渠道归属，运营在 selection.js 中手工调整 closed.channels 下的 items 即可。

### 创意/卖点数据（creative.js）
按赛道对象结构手工维护，字段说明见文件头部注释。每个赛道包含：
- `metrics` 指标均值
- `sellingWords` 卖点词云（weight 越大字越大越居中）
- `scripts` 常见话术
- `keyPoints` 跑量关键点（4维度）
- `topMaterials` Top素材（黄金五段式拆解）

### 节气日历（calendar.js）
年度节点已内置（24节气+大促+气候期+节日）。运营可在各节点 `items` 数组填充当期选品。

## 多人协作施工方案（方案二 · 腾讯文档联动）

考虑到各赛道行业运营**均非技术同学**，本项目采用【方案二：腾讯文档在线协作 + 定时同步】。
核心原则：**运营只在腾讯文档里填基础信息，页面逻辑与配图由系统自动完成**，运营不碰任何代码。

### 协作流程（运营视角，零门槛）

```
腾讯文档（多人在线填表）  →  导出 xlsx  →  同步脚本  →  data/*.js  →  页面自动刷新
   ↑ 行业运营只做这一步                        ↑ 由维护同学/自动化执行
```

1. **建两张腾讯文档在线表格**（多人可同时编辑，天然无冲突）：

   **表A · 选品填报表**（对应 `selection.js`）
   | 字段 | 说明 | 是否必填 |
   |------|------|:--:|
   | 链路 | 非闭环CID / 全域通 / ADQ | ✔ |
   | 二级赛道 | 如清洁工具、收纳、家纺… | ✔ |
   | 商品名 | 完整商品名称 | ✔ |
   | 叶子类目 | 更细分类目 | ✔ |
   | 客单价 | 数字 | ✔ |
   | ROI / CTR / CVR | 投放指标 | 选填 |
   | 落地页链接 | 小程序/CID链接 | 选填 |
   | 素材链接 | 视频/图文素材 URL | 选填 |

   > **商品图不用填**：系统会按「商品名」自动联网生成参考图（pollinations），运营无需手动补图。

   **表B · 创意卖点填报表**（对应 `creative.js`）
   | 字段 | 说明 |
   |------|------|
   | 赛道 | 对齐 5 个赛道 key |
   | 卖点词 / 权重 | 词云用，权重 0~100 |
   | 常见话术 | 一行一条 |
   | 跑量关键点 | 钩子/节点/福利/明星 四维度 |
   | Top素材 | 标题、产品、指标、原片URL、黄金五段式描述 |

   > **关键帧不用填**：给一条广告视频原片，由系统自动拆帧（见「视频拆帧」），运营只需维护文字信息。

2. **各赛道同学在腾讯文档里认领自己赛道的行**，各自填写，实时保存、多人并行。

3. **维护同学定时同步**（每周/每次发布前一次）：
   - 从腾讯文档导出 xlsx
   - 跑同步脚本转成 `data/*.js`（选品用 `tools/build_selection_data.py`，创意脚本同理）
   - 页面刷新即为最新对客版本

### 冲突最小化约定
- **每个赛道一个负责人（owner 字段已预留）**，只填自己赛道的行。
- 运营只填「基础信息（商品名/卖点词/指标/链接）」，**图片、关键帧、页面结构全部由系统/维护同学负责**。
- 商品图 / 素材 / 落地页统一用外链 URL 或自动生成，不往项目塞二进制文件。

### 视频拆帧（关键帧自动化）
给一条广告视频原片，即可自动按黄金五段式抽取 6 张竖屏关键帧，放入 Top 素材展示：
```bash
python3 tools/extract_frames.py   # 修改脚本内视频路径后运行，输出到 assets/frames/
```
